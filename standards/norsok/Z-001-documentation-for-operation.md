# NORSOK Z-001 — Documentation for Operation

## Scope
NORSOK Z-001 defines requirements for documentation to be delivered for the operation and maintenance of petroleum installations. It covers document types, content requirements, format, numbering, revision control, handover procedures, and management of as-built documentation. The standard ensures that operators receive complete, accurate, and usable documentation packages (DFO) throughout the project and facility lifecycle.

## Key Sections for Sifab AS
| Section | Topic | Relevance |
|---------|-------|-----------|
| §5 | General requirements | Document numbering, revision control, and format requirements applicable to all Sifab deliverables |
| §6 | Document register | Master Document Register (MDR) requirements — Sifab must maintain and submit an MDR for all project documentation |
| §7 | Engineering documents | Requirements for drawings, data sheets, calculations, and specifications — core Sifab engineering deliverables |
| §8 | Vendor documentation | Vendor Data Requirements (VDR) for sub-supplied equipment — flow meters, instruments, valves, analyzers |
| §9 | As-built documentation | As-built updating after fabrication and installation — red-line markups, final revision status |
| §10 | Operation and maintenance manuals | O&M manual requirements for metering systems, provers, tank gauging, and skid packages |
| §11 | Electronic documentation | Electronic format and delivery requirements — PDF, native CAD files, structured data |
| §12 | Handover | Documentation handover from project to operations — completeness checklists and punch list close-out |

## SifabAS Agent Usage
- **Primary**: Project Manager — MDR management, document delivery schedule, handover coordination, and DFO completeness verification
- **Secondary**: Quality Management Agent — document control per B.SI.01.07, ensuring all deliverables meet Z-001 requirements; ISO Document Producer Agent — formatting and numbering of internal QMS documents aligned with Z-001 conventions; all discipline agents — document format and content requirements for their respective deliverables
